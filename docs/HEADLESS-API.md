# Headless CMS Public API — Integration Guide

## Overview

The BlogCMS Headless API lets external applications (e-commerce sites, mobile apps, portals) consume blog content via a secure, rate-limited REST API. Each integration receives a dedicated API key scoped to a specific site.

## Authentication

All requests require a Bearer token in the `Authorization` header:

```
Authorization: Bearer bk_live_a1b2c3d4e5f6...
```

API keys are generated by an admin in the BlogCMS dashboard at **Admin → Integrations → New API Key**.

## Base URL

```
https://your-blogcms-domain.com/blog/api/v1
```

## Endpoints

### GET /api/v1/posts

List published posts scoped to your API key's site.

**Query Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | int | 1 | Page number |
| `limit` | int | 20 | Items per page (max 100) |
| `category` | string | — | Filter by category slug |
| `tag` | string | — | Filter by tag slug |
| `q` | string | — | Full-text search |

| `fields` | string | — | Comma-separated sparse fieldset (e.g. `title,slug,excerpt`) |

### GET /api/v1/posts/:slug

Get a single post by slug. Increments the view count.

### GET /api/v1/categories

List all categories for your site.

### GET /api/v1/tags

List all tags for your site.

### GET /api/v1/authors

List all authors who have published posts on your site.

## Rate Limiting

Each API key has a configurable rate limit (default: 60 requests/minute). Response headers:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 42
X-RateLimit-Reset: 1710590400
```

Exceeding the limit returns `429 Too Many Requests` with a `Retry-After` header.

## Post Visibility

Posts have a `visibility` setting:

- **public** — appears on the BlogCMS public feed only
- **external** — available via API only (does NOT appear on BlogCMS)
- **both** — appears everywhere

External-only posts are ideal for white-label content that should only appear on your site.

## Webhooks

Webhooks notify your app when posts are published, updated, or deleted. Configure webhooks in the API key detail page.

**Payload:**
```json
{
  "event": "post.published",
  "timestamp": "2026-03-21T10:00:00.000Z",
  "siteId": "uuid",
  "post": {
    "id": "uuid",
    "slug": "new-post",
    "title": "New Post Title",
    "publishedAt": "2026-03-21T10:00:00.000Z"
  }
}
```

**Verification:** Payloads are signed with HMAC-SHA256. Verify the `X-BlogCMS-Signature` header:

```javascript
import { createHmac } from 'crypto';

function verifySignature(payload, signature, secret) {
  const expected = createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  return expected === signature;
}
```

## Code Examples

### Vanilla fetch

```javascript
const API_KEY = process.env.BLOGCMS_API_KEY;
const BASE_URL = 'https://your-blogcms.com/blog/api/v1';

const response = await fetch(`${BASE_URL}/posts?limit=10`, {
  headers: { Authorization: `Bearer ${API_KEY}` },
});
const { data, pagination } = await response.json();
```

### Next.js (App Router)

```tsx
// app/blog/page.tsx
async function getPosts() {
  const res = await fetch(`${process.env.BLOGCMS_API_URL}/posts`, {
    headers: { Authorization: `Bearer ${process.env.BLOGCMS_API_KEY}` },
    next: { revalidate: 60 }, // ISR: revalidate every 60s
  });
  return res.json();
}

export default async function BlogPage() {
  const { data: posts } = await getPosts();
  return (
    <div>
      {posts.map((post) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <div dangerouslySetInnerHTML={{ __html: post.contentHtml }} />
        </article>
      ))}
    </div>
  );
}
```

### TanStack Start

```tsx
// routes/blog/index.tsx
import { createFileRoute } from '@tanstack/react-router';

async function fetchPosts() {
  const res = await fetch(`${import.meta.env.VITE_BLOGCMS_API_URL}/posts`, {
    headers: { Authorization: `Bearer ${import.meta.env.VITE_BLOGCMS_API_KEY}` },
  });
  return res.json();
}

export const Route = createFileRoute('/blog/')({
  loader: () => fetchPosts(),
  component: BlogPage,
});

function BlogPage() {
  const { data: posts } = Route.useLoaderData();
  return posts.map((post) => <h2 key={post.id}>{post.title}</h2>);
}
```

## SEO Guidance

1. **Server-side rendering is required.** Fetch posts in a server component, `getServerSideProps`, `loader`, or equivalent. Client-side-only fetch = no SEO.

2. **SSG + webhook revalidation** is recommended. Generate static HTML at build time, revalidate on `post.published` webhook for best performance + SEO.

3. **Canonical tags**: External-only posts have no URL on BlogCMS. No canonical tag needed. If a post has `visibility=both`, set `<link rel="canonical" href="https://blogcms.example.com/post-slug" />` to avoid duplicate content.

4. **Structured data**: The API response includes enough fields to generate `Article` JSON-LD schema.

5. **Cache headers**: API responses include `Cache-Control: public, max-age=60, stale-while-revalidate=300`.

## Error Responses

```json
{ "error": "Invalid API key" }           // 401
{ "error": "API key has been revoked" }   // 401
{ "error": "Rate limit exceeded" }        // 429
{ "error": "Post not found" }             // 404
```
